# Экспертная система профориентации

Веб-приложение для определения профессиональных склонностей на основе ответов на вопросы.

## Описание

Это оболочка экспертной системы, которая помогает определить подходящую профессию на основе:
- Интересов человека
- Особенностей характера
- Предпочтений в работе
- Готовности к обучению

Система использует **логический вывод** на основе правил и весовых коэффициентов для определения соответствия профессиям.

## Возможности

### 1. Прохождение теста
- 10 вопросов с 4 вариантами ответа:
  - Точно нет (0 баллов)
  - Скорее нет (1 балл)
  - Скорее да (2 балла)
  - Да (3 балла)
- Получение топ-5 рекомендаций профессий с процентом соответствия

### 2. Редактор базы знаний
- **Управление вопросами**: добавление, редактирование, удаление
- **Управление профессиями**: CRUD операции для профессий
- **Управление правилами**: настройка весов влияния вопросов на профессии

### 3. Управление базами знаний
- **Создание новой базы**: создание пустой базы знаний с нуля
- **Загрузка базы**: импорт базы знаний из JSON файла
- **Скачивание базы**: экспорт текущей базы знаний в JSON файл
- **Статистика**: просмотр количества вопросов, профессий и правил

### 3. Логический вывод
- Взвешенная система подсчета баллов
- Учет положительных и отрицательных факторов
- Процентное соответствие профессиям

## Технологии

- **Backend**: Python 3 + Flask
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla)
- **База знаний**: JSON
- **Логический вывод**: Собственный движок на Python

## Архитектура базы знаний

### Вопросы
```json
{
  "id": "q1",
  "text": "Текст вопроса",
  "answers": ["Точно нет", "Скорее нет", "Скорее да", "Да"]
}
```

### Профессии
```json
{
  "id": "programmer",
  "name": "Программист",
  "description": "Описание профессии"
}
```

### Правила
```json
{
  "question_id": "q1",
  "professions": {
    "programmer": 2,    // положительный вес
    "manager": 1,
    "designer": -1      // отрицательный вес
  }
}
```

## Логика работы

1. **Сбор ответов**: Пользователь отвечает на 15 вопросов
2. **Применение правил**: Для каждого ответа применяются правила из базы знаний
3. **Подсчет баллов**: 
   - Балл ответа (0-3) умножается на вес профессии в правиле
   - Суммируются баллы по всем вопросам для каждой профессии
4. **Вычисление процента**: 
   - Процент = (набранные баллы / максимально возможные) × 100%
5. **Ранжирование**: Профессии сортируются по проценту соответствия

## API Endpoints

### Получение данных
- `GET /api/questions` - Список вопросов
- `GET /api/professions` - Список профессий
- `GET /api/rules` - Список правил
- `GET /api/knowledge-base` - Вся база знаний

### Анализ
- `POST /api/analyze` - Анализ ответов и получение рекомендаций

### Управление базами знаний
- `POST /api/knowledge-base/create-new` - Создать новую пустую базу знаний
- `POST /api/knowledge-base/upload` - Загрузить базу знаний из JSON
- `GET /api/knowledge-base/download` - Скачать текущую базу знаний

### CRUD операции
- `POST /api/questions` - Добавить вопрос
- `PUT /api/questions/<id>` - Обновить вопрос
- `DELETE /api/questions/<id>` - Удалить вопрос

- `POST /api/professions` - Добавить профессию
- `PUT /api/professions/<id>` - Обновить профессию
- `DELETE /api/professions/<id>` - Удалить профессию

- `POST /api/rules` - Добавить правило
- `PUT /api/rules/<question_id>` - Обновить правило
- `DELETE /api/rules/<question_id>` - Удалить правило

## Демонстрация

### Пример использования:

1. Откройте `http://localhost:5000`
2. Ответьте на 10 вопросов
3. Нажмите "Получить результаты"
4. Просмотрите топ-5 рекомендованных профессий

### Редактирование базы знаний:

1. Перейдите на вкладку "Редактор базы знаний"
2. Добавьте новый вопрос или профессию
3. Настройте правила (веса влияния)
4. Изменения сохраняются автоматически в `knowledge_base.json`
