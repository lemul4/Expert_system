# Экспертная система профориентации

Веб-приложение для определения профессиональных склонностей, построенное на базе классической теории экспертных систем (MYCIN).

## Описание

Это оболочка экспертной системы, которая использует **Коэффициенты Уверенности Шортлиффа (Certainty Factors, CF)** для моделирования рассуждений в условиях неопределенности. В отличие от простых тестов с суммированием баллов, эта система оценивает степень уверенности в гипотезах, накапливая доказательства «За» и «Против» каждой профессии.

Система учитывает:
- Силу уверенности пользователя в ответе.
- Вес (силу) правила, заданную экспертом.
- Эффект накопления знаний (асимптотическое приближение к 100%).
- Разрешение конфликтов (когда одни факты говорят "Да", а другие "Нет").

## Теоретическая база: Метод Шортлиффа

В основе движка лежит математический аппарат, разработанный для медицинской системы **MYCIN**.

1. **Коэффициент уверенности (CF)**: Число в диапазоне `[-1.0; +1.0]`.
   - `+1.0`: Абсолютная уверенность (Истина).
   - `-1.0`: Абсолютное опровержение (Ложь).
   - `0.0`: Отсутствие информации.
2. **Агрегация знаний**: Система использует формулу Шортлиффа для объединения свидетельств. Два слабых аргумента "За" усиливают друг друга, но никогда не переходят границу 100%.
3. **Обработка противоречий**: Если данные противоречат друг другу, система взвешивает их, уменьшая итоговую уверенность.

## Возможности

### 1. Прохождение теста
- 10 ключевых вопросов, направленных на выявление психотипа и интересов.
- Интерпретация ответов пользователя в Evidence CF:
  - **Точно нет** → CF -1.0
  - **Скорее нет** → CF -0.5
  - **Скорее да** → CF +0.5
  - **Да** → CF +1.0
- Фильтрация "шума": слабые сигналы (CF < 0.2) игнорируются.

### 2. Редактор базы знаний
- **Управление вопросами**: CRUD операции.
- **Управление профессиями**: Добавление и описание целевых гипотез.
- **Настройка правил (Весов)**: Установка силы связи от `-1.0` (строго не подходит) до `1.0` (идеально подходит).

### 3. Управление базой данных
- **Импорт/Экспорт**: Загрузка и сохранение базы знаний (`knowledge_base.json`).
- **Сброс**: Создание новой пустой базы.

## Технологии

- **Backend**: Python 3 + Flask
- **Core Logic**: Собственная реализация Inference Engine (Certainty Factors).
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla).
- **Storage**: JSON (файловое хранение).

## Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Запуск приложения

```bash
python app.py
```

Приложение будет доступно по адресу: `http://localhost:5000`
